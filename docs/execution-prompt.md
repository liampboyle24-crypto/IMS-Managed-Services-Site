# Claude Code Execution Prompt

Use this prompt in Claude Code web after opening this GitHub repo.

---

Read `CLAUDE.md`, `docs/design-brief.md`, and `index.html`.

I want you to take a serious professional redesign pass on this single-page website. Do not make shallow cosmetic tweaks. The goal is a polished, credible, partner-facing B2B technology website.

Use the available skills selectively and explicitly:

* Use `/frontend-design` for layout, typography, hierarchy, responsive behavior, and frontend polish.
* Use `/emil-design-eng` for design engineering quality and thoughtful implementation.
* Use `/taste-skill` as a final taste and restraint pass.
* Use `/redesign-skill` only where sections need to be rebuilt instead of lightly edited.
* Use `/review-animations` only if subtle interaction improvements are useful.
* Avoid `/brutalist-skill` unless you have a very specific and restrained reason.

Workflow:

1. Audit the current page before editing.
2. Identify design, messaging, typography, layout, responsiveness, and credibility issues.
3. Create a concise revision plan.
4. Implement the improvements directly in `index.html`.
5. Improve the mobile experience.
6. Tighten copy where it improves clarity, but do not invent unsupported claims.
7. Remove anything that feels generic, cluttered, or AI-generated.
8. Make the final page feel premium, professional, and partner-ready.
9. Verify that the page still works as a static HTML file.
10. Summarize what changed and what should be reviewed manually.

Important constraints:

* Keep `lboyle@imsts.com` visible.
* Remove references to 2026 unless required by approved copy.
* The top CTA should scroll to the contact section, not imply a separate assessment tool unless one exists.
* Prioritize clarity, trust, polish, and conversion.
* Do not add unnecessary dependencies.
* Do not invent fake awards, metrics, certifications, clients, or claims.

Before you finish, do a final taste pass and remove anything that looks overdesigned, trendy, inconsistent, or AI-generated.
