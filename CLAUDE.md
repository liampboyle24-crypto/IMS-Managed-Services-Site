# CLAUDE.md

## Project Mission

This repo contains a single-page HTML website that needs a high-end professional redesign and polish pass.

The goal is to transform `index.html` into a premium, credible, partner-facing B2B website suitable for a technology, AV, managed services, or systems integration audience.

The final result should feel:

* Executive-ready
* Premium but not gimmicky
* Clear, credible, and conversion-focused
* Modern without looking like generic AI-generated SaaS design
* Strong enough to send to partners, prospects, or internal leadership

## Primary File

Work primarily in:

* `index.html`

## Available Skills

Several design and frontend skills are available in `.claude/skills/`.
Use them selectively and intentionally. Prioritize high-level visual impact.
Smooth animations and dynamic elements are appreciated

## Design Standards

Prioritize:

* Clear section hierarchy
* Strong typography
* Clean spacing and alignment
* High contrast and legibility
* Premium visual rhythm
* Credible B2B messaging
* Clear calls to action
* Dynamic animations or transitions between sections
* Polished details that feel intentional
* Include image assets in references foler where appropriate

Avoid:

* Generic AI-generated visuals
* Overuse of gradients
* Excessive glassmorphism
* Low-contrast text
* Fake metrics, fake awards, or unsupported claims
* Decorative elements that make the page harder to read
* Bloated libraries or unnecessary dependencies
* Design trends that will age poorly

## Content Guardrails

Do not invent company claims, certifications, metrics, awards, client names, or capabilities that are not already present or provided.

Do not mention 2026 unless it already appears in approved source copy and is clearly required.

Make sure this email is visible on the page:

[lboyle@imsts.com](mailto:lboyle@imsts.com)

If there is a top CTA, it should scroll to the contact section. Do not imply a separate assessment tool, form, or workflow unless one actually exists on the page.

## Workflow

Before editing:

1. Read this file.
2. Inspect `index.html`.
3. Identify the strongest and weakest sections.
4. Create a revision plan.

Then:

1. Improve the visual design.
2. Tighten messaging where needed.
3. Improve typography, spacing, layout, and visual hierarchy.
4. Improve mobile responsiveness.
5. Remove or replace anything that feels generic, cluttered, or AI-generated.
6. Preserve factual accuracy.
7. Keep the site easy to open as a static HTML file.

After editing:

1. Summarize what changed.
2. Explain which skills were used and why.
3. Note anything the user should review manually.
4. Confirm the page still works as a static HTML file.
